## 基本流程

```shell
# 初始化仓库
git init
# 将本地库关联至远程仓库
git remote add origin git@github.com:....github.io.git
# 查看当前修改状态 
git status 
# 添加修改过得文件， . 表示所有，也可以指定文件 
git add . 
# ""里面的内容就是提交内容的说明信息 
git commit -m "first commit"

# 第一次提交方法1
git push -u -f origin main

#第一次提交方法2
git pull origin main --allow-unrelated-histories 
git push -u origin main

# 以后提交
git push
```



## 其他用法

### 1 修改分支名

```shell
git branch -m oldBranchName newBranchName
```

### 2 取消与远程仓库的关联

```shell
git remote remove origin
```

### 3 实现本地库同时关联GitHub和Gitee

```shell
# 初始化仓库
git init
# 将本地库同时和GitHub、Gitee的远程仓库关联
git remote add github git@github.com:bertilchan/gitTest.git
git remote add gitee git@gitee.com:bertil/git-test.git
# 查看关联的远程库信息
git remote -v
# 添加修改，和原来一样
git add .
git commit -m "first commit"
# 分别提交
git push -u github main
git push -u gitee main
```

### 4 Github嵌套项目拉取

```shell
# 获取主项目和所有子项目源码
git clone --recurse-submodules <main_project_url>  
```

### 5 从远程拉取所有分支

```shell
# git clone xxx
git branch -r | grep -v '\->' | while read remote; do git branch --track "${remote#origin/}" "$remote"; done
git fetch --all
git pull --all
```
